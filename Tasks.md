## M6: メインタブ差分の視覚的強調（半透明オーバーレイ）



### M6-001: 差分マスク生成ユーティリティ

- [ ] 二枚の RGBA `Image` としきい値等からマスク（boolean または uint8）を返す関数を `utils/` に切り出す。  

      **確認観点**: `numpy` と `PIL` のみで完結し、`apply_color_processing_to_image` 適用後の画像を想定した入出力が明確であること。  

      → チャネル差（RGB・A）に基づくしきい値、必要ならノイズ抑制（最小面積や単純モフォロジー）を関数境界で分離する。

- [ ] 入力サイズ不一致時の扱いを決め、実装するか明示的に例外とする。  

      **確認観点**: メインタブの `_display_page` で得られる base/comp の寸法関係と矛盾しないこと。  

      → 単体テストまたは最小再現スクリプトでマスク形状が期待どおりであることを確認する。



### M6-002: メインタブへのオーバーレイ統合

- [ ] 「差分強調」トグル（チェックまたはトグルボタン）を配置し、状態を保持する。  

      **確認観点**: トグル OFF 時に `diff_emphasis` 系のキャンバス要素が残らないこと。  

      → `views/main_tab.py` の UI 構築箇所に追加し、テーマ色が崩れないようにする。

- [ ] `_display_page` の描画フローに合わせ、トグル ON かつベース／比較の画像が両方利用可能なときだけオーバーレイを生成・表示する。  

      **確認観点**: 変形適用後の `base_image` / `comp_image` と同一座標系で貼り付けられること。  

      → PIL で半透明 RGBA を生成し、`canvas.create_image(..., tags=("diff_emphasis",))` と `tag_raise` で既存レイヤーより手前にする。

- [ ] ページ変更、レイヤー表示 ON/OFF、変形、色処理モード、各色・しきい値、プレビュー再読み込みのタイミングでオーバーレイが更新または削除される。  

      **確認観点**: 古いページのマスクが残らないこと。  

      → 必要に応じキャッシュキーにページインデックス・変形・色設定を含める。
