# タスクリスト

## M4: ライセンス整理と説明/ライセンスタブ整備

### M4-001: ランタイム依存ライセンスの棚卸し
- [ ] `pyproject.toml` / `uv.lock` / `uv tree --no-dev` を基準に、ランタイム依存一覧を固定する。  
      **確認観点**: 開発依存とランタイム依存が混在していないこと。  
      → `svglib` が `LGPL-3.0-or-later`、その配下で `pycairo` が `LGPL-2.1-only OR MPL-1.1` である点を配布レビュー対象として整理する。
- [ ] 依存ライセンスのうち、配布判断が必要なものを明示する。  
      **確認観点**: permissive / reciprocal の区分が一目で分かること。  
      → `licences_tree.txt` では review 推奨マーカーを使い、`licences.txt` では現行監査結果を先頭へ置く。

### M4-002: ライセンス文書の整備
- [ ] `licences_tree.txt` を現行依存ツリーに合わせて更新する。  
      **確認観点**: `reportlab` / `svglib` / 下位依存が反映されていること。  
      → `uv tree --no-dev` の結果を基準に更新する。
- [ ] `licences.txt` を現行の監査結果が先頭で読める構成へ更新する。  
      **確認観点**: ランタイム依存の現状が旧スナップショットより先に読めること。  
      → 旧本文は参考用に残しつつ、最新監査ブロックを先頭へ追加する。

### M4-003: ライセンスタブの表示整備
- [ ] `views/licenses.py` を `licences_tree.txt` / `licences.txt` 読込型へ置き換える。  
      **確認観点**: 静的MIT/GPL文ではなく、ワークスペース内ファイルが表示されること。  
      → 概要/詳細ファイルを開くボタンと再読込ボタンを持つ構成にする。
- [ ] ライセンスタブへカラーモードを適用する。  
      **確認観点**: dark / light / pastel で本文・背景・ボタンの視認性が保たれること。  
      → `Frame` / `Label` / `text_box` / `process_button` 系テーマ色で統一する。

### M4-004: 説明タブの内容更新
- [ ] `docs/milestone/*PLAN.md` を基に、各タブの運用手順を説明タブへ掲載する。  
      **確認観点**: Main / PDF操作 / U006 / Licenses の操作方針が読めること。  
      → M1 / M2 / M2.1 / M3 の見出しを参照元として併記する。
- [ ] 説明タブのボタンにもカラーモードを適用する。  
      **確認観点**: temp / logs / debug.log を開くボタンが各テーマで崩れないこと。  
      → 説明文本文は `text_box`、操作ボタンは `process_button` 系で揃える。

### M4-005: ライセンス方針の次アクション整理
- [ ] `svglib` を継続利用するか、代替実装へ寄せるかの判断材料を残す。  
      **確認観点**: 次回着手時に「何をレビューすべきか」が分かること。  
      → LGPL を許容する配布形態、または SVG 対応の縮小/代替案を候補として明記する。
- [ ] 検証手順を整理する。  
      **確認観点**: UI確認と文書確認の両方を再現できること。  
      → タブ表示、ファイル読込、テーマ切替、ライセンス注意表示を手順化する。
