# M2.1 Plan — U006追加仕様（PDF解像度・用紙サイズ・文言統一）

## 目的
- M2（ファイル拡張子・サイズタブ, U006）の仕様誤解を補正し、PDFの実運用要望を正式仕様として定義する。
- 特に以下をM2.1として明文化する。
  1. PDF入力時のDPI変更（主に軽量化目的のダウンサンプリング）
  2. 用紙サイズ（物理サイズ）を意識した変換
  3. 出力単位に応じた `file` / `folder` 文言の統一

## スコープ
- 対象タブ: 「ファイル拡張子とサイズ」タブ（U006）
- 対象機能: 拡張子変換（M2-003）を中心に、PDF関連変換仕様を拡張
- 非対象:
  - PDF操作タブ（別タブ機能）
  - 既存のコピー保護判定仕様（M2-005で定義済み）

## 背景整理
- 現行M2仕様では、PDF→画像は pypdfium2 のラスタライズで処理する。
- 一方で、実利用では「画質を維持する上げ方向」だけでなく「軽量化のためのDPI下げ方向」要件がある。
- また、紙面サイズ（A4等）ベースで入出力を扱いたい要望がある。

## 追加仕様（M2.1）

### 1) PDF入力時のDPI指定（M2-003拡張）
- PDF→画像変換時、ラスタライズDPIをユーザー指定可能とする。
- 既定値は 300dpi とし、72/96/150/300/600 + カスタム入力を許可する。
- 指定DPIに応じてラスタライズ倍率を変更する。
- 低DPI指定時は「軽量化優先（画質低下の可能性あり）」警告を表示可能とする。

### 2) 用紙サイズ（物理サイズ）対応
- PDFページの物理サイズ（pt）を保持し、`px = pt × dpi / 72` で出力ピクセルを決定する。
- ユーザーが用紙サイズを指定した場合は、指定サイズを優先し、縦横方向（Portrait/Landscape）を維持する。
- 用紙サイズ指定がない場合は、元PDFページサイズを基準に出力する。

### 3) file / folder 文言の統一（M2-003へ正式反映）
- 単一出力: `file` 文言を使用する。
- 複数ページ/複数フレーム出力: 専用出力ディレクトリを作成し `folder` 文言を使用する。
- ステータスバー、警告ラベル、確認ダイアログの3経路で同一語彙を使用する。
- 重複回避の語彙も出力単位に合わせる（file重複 / folder重複）。

## UI要件（U006内）
- PDF入力時にのみ有効な「PDF出力DPI」入力欄を拡張子変換ブロックへ表示する。
- 既存DPIドロップダウンの流用可。ただしPDF入力時の意味（ラスタライズDPI）をラベルで明示する。
- メタ情報欄に以下を表示する。
  - Format: PDF
  - Page size: `W×H pt`
  - Page count
  - Effective raster DPI（実際に使用したDPI）

## 受け入れ基準
- PDF→画像時、DPIを下げるとファイルサイズが低減する。
- 同一PDFでDPIを変更した場合、出力px寸法が変化する。
- 複数ページPDFは常にfolder出力となり、文言がfolderで統一される。
- 単一ページ出力はfile文言が使われる。

## 実装メモ
- PDFラスタライズは pypdfium2 の `render(scale=...)` を使用し、`scale = dpi / 72` とする。
- 既存の重複回避ロジックを流用し、単一/複数出力で file と folder を分岐する。
- メッセージコードは既存を再利用し、不足分のみ追加する。
