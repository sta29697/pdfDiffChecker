# M2.1 Plan — U006追加仕様（PDF解像度・用紙サイズ・文言統一）

## 目的
- M2（ファイル拡張子・サイズタブ, U006）の仕様誤解を補正し、PDFの実運用要望を正式仕様として定義する。
- 特に以下をM2.1として明文化する。
  1. PDF入力時のDPI変更（主に軽量化目的のダウンサンプリング）
  2. 用紙サイズ（物理サイズ）を意識した変換
  3. 出力単位に応じた `file` / `folder` 文言の統一

## スコープ
- 対象タブ: 「ファイル拡張子とサイズ」タブ（U006）
- 対象機能: 拡張子変換（M2-003）を中心に、PDF関連変換仕様を拡張
- 非対象:
  - PDF操作タブ（別タブ機能）
  - 既存のコピー保護判定仕様（M2-005で定義済み）

## 背景整理
- 現行M2仕様では、PDF→画像は pypdfium2 のラスタライズで処理する。
- 一方で、実利用では「画質を維持する上げ方向」だけでなく「軽量化のためのDPI下げ方向」要件がある。
- また、紙面サイズ（A4等）ベースで入出力を扱いたい要望がある。

## 追加仕様（M2.1）

### 1) PDF入力時のDPI指定（M2-003拡張）
- PDF→画像変換時の有効DPIは、以下の優先順位で決定する。
  1. 入力PDFから検出できたmeta DPI（`X/Y` 乖離の有無にかかわらず採用）
  2. meta情報がない場合のみ、ページpt値とレンダリングpixel値からの計算DPI
  3. 計算DPIが不採用の場合は、既定値 300dpi を使用（必要に応じてユーザーがドロップダウンで変更）
- meta DPI が取得できた場合は、その値を正として扱い、`X/Y` 差分による棄却は行わない。
- 計算DPIは `meta情報がない場合` のみ評価し、`X/Y` 乖離率が閾値（初期値 10%）を超える場合は不採用とする。
- 計算DPIが不採用または算出不能の場合、初期値300dpiを適用し、拡張子変換ブロックの `PDF Raster DPI` ドロップダウンでユーザーが値を選択できるようにする。
- 決定した有効DPIに応じてラスタライズ倍率を変更する。
- 低DPI時は下記文言で確認警告を必ず表示する。  
  `指定したラスタライズDPI（{dpi}）は推奨値300dpi未満です。文字や細線の視認性が低下する可能性があります。続行しますか？`

### 2) 用紙サイズ（物理サイズ）対応
- PDFページの物理サイズ（pt）を保持し、`px = pt × dpi / 72` で出力ピクセルを決定する。
  - `72` は固定定数（1 inch = 72 pt）であり、メタ情報/計算結果/既定値の違いで変化しない。
  - 変化するのは `dpi` の参照元のみ（メタ情報 / 計算結果 / 既定値）。
- ユーザーが用紙サイズを指定した場合は、指定サイズを優先し、縦横方向（Portrait/Landscape）を維持する。
- 用紙サイズ指定がない場合は、元PDFページサイズを基準に出力する。

### 2.1) Resize入力方式の変更（px指定廃止 → 倍率指定）
- Resizeブロックの幅/高さ `px` 手入力は廃止し、幅倍率・高さ倍率の入力方式へ変更する。
- 倍率は `1.0 = 100%` を基準とし、幅倍率・高さ倍率を独立指定できるようにする。
- 縦横比固定が有効な場合は、一方の倍率変更時に他方も同倍率へ同期する。
- 現在サイズ表示は代表値ではなく、「現在サイズ」と「適用予定倍率」を併記し、変換後の見込みサイズが追えるUIとする。
- 用紙サイズ優先モードと倍率指定は排他的に扱い、どちらを優先しているかをUI上で明示する。

### 3) file / folder 文言の統一（M2-003へ正式反映）
- 単一出力: `file` 文言を使用する。
- 複数ページ/複数フレーム出力: 専用出力ディレクトリを作成し `folder` 文言を使用する。
- ただし以下の統合可能ケースは、複数ページ入力でも単一 `file` 出力とする。
  - PDF → TIF（multi-page TIFF）
  - TIF → PDF（multi-page PDF）
- ステータスバー、警告ラベル、確認ダイアログの3経路で同一語彙を使用する。
- 重複回避の語彙も出力単位に合わせる（file重複 / folder重複）。

### 4) Resizeブロックの複数ページ対応方針（更新版）
 - Resizeブロックの複数ページ対応は、**同サイズの複数ページ入力のみ** を対象とする。
 - `PDF` の複数ページ入力では、全ページのサイズ一致を事前確認し、一致した場合のみ倍率指定による一括Resizeを許可する。
 - `multi-frame TIF` を含む **multi-frame入力は対象外** とし、Resizeブロックでは引き続き非アクティブ表示とする。
 - サイズ不一致の複数ページ入力は対象外とし、代表値ベースのまま変換を続行させない。
 - 対象外ケースでは、なぜ無効なのかを warning / dialog の両方で明示する。
 - 同サイズ複数ページを許可する際に必要な確認ダイアログは、以下の条件で再整理する。
   - ページサイズ一致確認結果を表示する。
   - 回転や向きの差異があれば、Resize前に調整要否を確認する。
   - DPIの扱いに候補差がある場合は、採用基準値を明示して確認する。
 - 負荷抑制のため、処理ページ上限を内部変数で管理する（初期値 20）。
 - 上限超過時は処理前に警告し、上限値変更が可能な設計（定数/設定値）とする。

## UI要件（U006内）
- 拡張子変換ブロックの `PDF Raster DPI` は、meta情報がない かつ 計算DPIが不採用/算出不能の場合に表示・有効化する（初期値300dpi）。
- Resizeブロックのサイズ指定UIは、幅/高さ `px` 欄ではなく、幅倍率・高さ倍率入力欄で構成する。
   - 初期値は `1.0 / 1.0` とする。
   - 縦横比固定が有効な場合は、一方を変更すると他方も同期する。
- Resizeブロックの「Output DPI」は、入力ファイル種別に応じて以下の状態とする。
  - 有効化条件: `PDF` または `TIF` 入力
  - 無効化条件: `PDF` / `TIF` 以外の入力
  - 既存UIで選択可能な入力形式は減らさず維持する
- Resizeブロックの `Output DPI` は、サイズ変換時に使用するDPI値そのものを表示し、`PDF` / `TIF` 入力時はユーザーが変更できるようにする。
  - `PDF` 入力時の初期値は、meta情報が取得できればその値、取得できなければ計算結果、さらに算出不能時は既定値300dpiとする
  - `TIF` 入力時の初期値は、画像DPIが取得できればその値、取得できなければ既定値300dpiとする
  - `JPEG` / `SVG` を含む `PDF` / `TIF` 以外の入力時は、表示値は参考表示のみとし、編集は不可とする
 - メタ情報欄に以下を表示する。
   - Format: PDF
   - Page size: `W×H pt`
   - Page count
   - Effective raster DPI（実際に使用したDPI）
   - DPI source（メタ情報 / ユーザー指定 / 既定値）
   - Calculation DPI（参考値。必要時のみ表示）
 - multi-page入力時のメタ情報欄は、1ページ目を代表値として表示し、代表表示である旨の注記をメタ情報表示領域内で常に視認できる形で表示する。
 - Resizeブロックの複数ページ対応状態は、以下の3値をUI上で区別できるようにする。
   - 同サイズ複数ページのため変換可能
   - サイズ不一致複数ページのため変換不可
   - multi-frame入力のため変換不可
 - サイズ変換で倍率が未入力または不正な場合は、path系メッセージではなく、倍率入力不足を明示する専用文言を表示する。

## 受け入れ基準
- PDF→画像時、DPIを下げるとファイルサイズが低減する。
- 同一PDFでDPIを変更した場合、出力px寸法が変化する。
- meta DPI が存在する場合は `X/Y` 差分があっても meta を採用する。
- meta情報がない場合のみ計算DPIを評価し、`X/Y` 乖離率が閾値超過なら不採用とする。
- 計算DPI不採用時は300dpiを初期値とし、ユーザーはドロップダウンからDPIを選択できる。
- 実効DPIが300未満となる変換では、低解像度警告（確認ダイアログ）が必ず表示される。
- 統合可能ケース（PDF→TIF, TIF→PDF）は複数ページ入力でも単一file出力となる。
- 統合不能ケースのみfolder出力となり、文言がfolderで統一される。
- Resizeブロックは幅倍率・高さ倍率入力で動作し、`1.0` 指定時は元サイズ相当となる。
- 同サイズ複数ページ入力のみ、倍率指定Resizeを許可する。
- サイズ不一致複数ページ入力では、Resizeブロックは変換不可表示となり、処理を続行できない。
- multi-frame入力では、Resizeブロックは変換不可表示となり、処理を続行できない。
- Resizeブロックでの複数ページ処理は、ページ上限（初期値20）以内で動作し、上限超過時は警告される。
- PDF→TIF / TIF→PDF は、一時フォルダ経由で単一ファイルへ統合出力できる。

## 実装メモ
- PDFラスタライズは pypdfium2 の `render(scale=...)` を使用し、`scale = dpi / 72` とする。
- Resize倍率は、各ページの元サイズに対する乗率として扱い、`output_px = round(input_px × scale)` で計算する。
- 同サイズ複数ページ判定は、全ページの `W×H` 一致確認を通過条件とする。
- multi-frame入力は判定段階でResize対象外として分岐する。
- 既存の重複回避ロジックを流用し、単一/複数出力で file と folder を分岐する。
- メッセージコードは既存を再利用し、不足分のみ追加する。
