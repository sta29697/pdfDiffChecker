# M4 Plan — ライセンス整理と説明/ライセンスタブ整備

## 目的
- ランタイム依存ライセンスの現状を整理し、配布判断が必要な項目を追跡できる状態にする。
- `licences_tree.txt` / `licences.txt` を現行依存に追従させ、ライセンスタブから参照可能にする。
- 説明タブの内容を現行運用へ合わせて更新し、各タブの操作意図と注意点を読めるようにする。

## 現状整理
- 詳細チェックリストは `docs/tasks/M4_PLAN.md` に集約する（旧リポジトリ直下 `Tasks.md` から移動）。
- 依存関係には `svglib` など配布レビューが必要なものが含まれており、単純な permissive-only 前提では整理できない。
- ライセンスタブと説明タブは、文書の最新内容と必ずしも同期していない可能性がある。

## スコープ
- 対象:
  - `licences_tree.txt`
  - `licences.txt`
  - `views/licenses.py`
  - `views/description.py`
  - `docs/milestone/*.md`
  - `docs/tasks/M4_PLAN.md`
  - `docs/tasks/M3_Tasks.md`
- 非対象:
  - 外部依存そのものの入れ替え
  - ライセンス方針の最終法務判断
  - リリースビルド手順の全面刷新

## 実装方針

### 1) ランタイム依存ライセンスの棚卸し
- `pyproject.toml` / `uv.lock` / `uv tree --no-dev` を基準に、ランタイム依存一覧を固定する。
- 配布判断が必要な依存について、レビュー対象であることが文書から分かる構成にする。
- `svglib` とその下位依存は、配布判断が必要な候補として明示する。

### 2) ライセンス文書の整備
- `licences_tree.txt` は現行依存ツリーに合わせて更新する。
- `licences.txt` は最新監査結果を先頭で読める構成に整理する。
- 旧来の参考情報が必要な場合は、最新監査結果を優先しつつ後段へ退避する。

### 3) ライセンスタブの表示整備
- `views/licenses.py` は静的な固定文言ではなく、ワークスペース内のライセンス文書を参照する構成へ寄せる。
- 概要文書と詳細文書の双方へアクセスできる導線を用意する。
- dark / light / pastel の各テーマで視認性が維持される配色に揃える。

### 4) 説明タブの内容更新
- `docs/milestone/*PLAN.md` を参照元として、各タブの運用手順を説明タブへ反映する。
- Main / PDF操作 / U006 / Licenses の各タブについて、用途・基本手順・注意点を整理する。
- 説明タブ内の操作ボタンも、現行テーマ基盤で崩れないようにする。

### 5) 次アクションの明文化
- `svglib` 継続利用か代替方針かの判断材料を残す。
- 今後の検証手順をまとめ、次回作業者が再現確認しやすい形にする。

## 継続利用判断の材料
- 現行ランタイム依存では `svglib` が SVG/PDF 出力経路に入り、下位依存として `cssselect2` / `tinycss2` / `webencodings` / `lxml` / `rlpycairo` / `freetype-py` / `pycairo` / `reportlab` を伴う。
- 技術監査上は、`svglib` と `pycairo` 系を含む経路を「配布レビュー要」として追跡する。
- 継続利用する場合は、実際の配布形態（同梱バイナリ、インストーラ、社内限定配布、ソース同梱有無）に対して、必要な表示義務・同梱義務・差分開示要否を法務/製品判断へ引き継ぐ。
- 代替方針を検討する場合は、まず SVG 対応の必要範囲を整理し、以下を比較候補にする。
  - SVG 入力をサポート対象外にして依存を削減する。
  - SVG を事前変換した成果物のみ受け入れる運用へ寄せる。
  - `svglib` 非依存の変換経路へ置き換える。

## 検証手順
1. `uv run main.py` でアプリを起動する。
2. `Licenses` タブを開き、`licences_tree.txt` の内容が初期表示されることを確認する。
3. 「ライセンス全文を表示」を押し、`licences.txt` の先頭に最新監査結果サマリが表示されることを確認する。
4. `Licenses` タブ上部の参照中パス表示が、現在のワークスペース文書パスを示すことを確認する。
5. テーマを dark / light / pastel に切り替え、本文・ラベル・ボタンの視認性が崩れないことを確認する。
6. `説明` タブを開き、Main / U006 / Licenses の運用説明と、`docs/milestone/*PLAN.md` の参照一覧が表示されることを確認する。
7. 説明タブ下部の一時保存先ボタン、ログ保存先ボタンの文言が言語切替とテーマ切替後も正しく維持されることを確認する。

## 実装メモ
- Python 実行系の確認には `uv run` を使う。

- UI文言更新と文書更新は、参照元の対応関係が分かるように揃える。
