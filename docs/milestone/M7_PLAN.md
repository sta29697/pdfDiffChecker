# M7 Plan — キーボード操作とフォーカス順（自動化・ショートカット）

## 目的
- マウスに頼らず、**タブ切替**・**メインタブ内の UI**・**プレビュー右のページ操作ブロック**をキーボードで辿れるようにする。
- 既存の**キャンバス用ショートカット**は、キャンバスにフォーカスがあるときはそのまま利用できるようにする（Ctrl+クリック相当の複合操作はキーボードでは代替しない）。

## 背景・現状整理
- メイン UI は `ttk.Notebook`（`main.py`）と各タブの `Frame` で構成される。メインタブは `CreateComparisonFileApp`（`views/main_tab.py`）がグリッドで多数のコントロールと `tk.Canvas`、および `PageControlFrame`（`widgets/page_control_frame.py`）を持つ。
- Tk の既定 Tab 順は作成順に依存し、列優先の視覚順と一致しないことが多い。
- `ttk.Combobox` は環境によって **Down でリストを開く**挙動が不安定なため、必要なら Tcl の `ttk::combobox::Post` で補助する。

## スコープ
- 対象（初版）:
  - **ノートブック**: Alt 単独で「タブストリップモード」に入り、**左右キー**でタブ切替。`Escape` でモード解除。
  - **メインタブ**: 列優先に近い **カスタム Tab / Shift+Tab** チェーン（言語・テーマ、パス行、しきい値・自動/カスタム実行、レイヤー・DPI・色モード、キャンバス、`PageControlFrame` 一式）。
  - **PageControlFrame**: ページ送り・数値入力・変形 Entry・エクスポートまでをチェーンに含める。
- 対象（拡張）:
  - **PDF 操作タブ**: 言語・テーマ、共有パス行（列優先）、プレビュー **Canvas**（`takefocus`）、読み込み後の `PageControlFrame` をチェーンに含める。
  - **画像操作タブ**: 言語・テーマ、パス行、拡張子変換ブロック、サイズ変換ブロックの主要コントロールをチェーンに含める。
- 非対象（明示的制約）:
  - **説明タブ / ライセンスタブ** は既定の Tk フォーカス順のまま（スクロールテキストの Tab を壊さない）。
  - キャンバス上の **Ctrl+クリック相当**はキーボードでは提供しない。

## 実装方針
1. **`KeyboardNavigationShell`**（`controllers/keyboard_navigation.py`）が `bind_all` で Alt 追跡、タブストリップモード、左右、Tab / Shift+Tab、Combobox の Down を処理する。
2. **メインタブ**は `build_keyboard_focus_chain()` で毎回チェーンを組み立てる（`PageControlFrame` の再生成に追随するため）。
3. **Canvas** は `takefocus=1` とフォーカス時の `highlightthickness` で、フォーカス位置が分かるようにする。
4. 説明タブ等の **スクロール可能テキスト**では、フォーカスがそのタブ内かつチェーン外のときは **既定の Tab 挙動**を壊さない（カスタムチェーンはメイン / PDF / 画像タブのみ返す）。

## 検証手順（目安）
1. メインタブで **Alt を単独**押下後、左右でタブが切り替わること。`Tab` でタブストリップモードが終わり、先頭付近のコントロールにフォーカスが移ること。
2. **Tab / Shift+Tab** で、パス入力→各ボタン→しきい値 Entry→チェック・コンボ→キャンバス→右サイドバーの順に進む／戻れること（無効ウィジェットはスキップ）。
3. 言語など **Combobox** にフォーカスがある状態で **Down** がリスト表示に繋がること（環境差は Tcl 呼び出し失敗時はスキップ）。
4. キャンバスにフォーカスしたうえで、既存の**矢印等のショートカット**が動作すること。
5. **PDF 操作タブ**で Tab が言語→テーマ→パス行→キャンバス→（PDF 読み込み後）ページ操作ブロックへ辿れること。**画像操作タブ**で拡張子・サイズブロックの主要コントロールまで辿れること。

## 実装メモ
- Windows で **Alt 単独**は OS のメニューバーアクセラレータと競合しうる。メニューバーが無い構成では通常問題にならないが、挙動がおかしい場合は `Escape` でモード解除できることと、左右でのタブ切替を確認する。
- リポジトリにコミットする **ユーザー設定のパス**はプレースホルダ（例: `C:\Users`）に揃え、個人パスを載せない。
