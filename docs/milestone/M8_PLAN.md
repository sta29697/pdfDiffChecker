# M8 Plan — リリース資産、コンボボックス矢印、差分強調の改善

## 目的

1. **GitHub Release に exe が付かなかった理由**を整理し、再発しにくい手順・設定にする。
2. **キーボード**: コンボボックスにフォーカスがあるとき、**Enter を押すまで**矢印キーでプルダウンを開かず、上下キーで隣接コントロールへフォーカスを移せるようにする（リストが既に開いているときは従来どおりリスト内移動）。
3. **メインタブの差分強調**: 囲み付近のように、**細い線・小さな字形差**が半透明オーバーレイでほとんど見えないケースを減らす。

## 背景・原因整理（exe と .gitignore）

- `.gitignore` で `build/` を広く無視しつつ `build/nuitka/pdfDiffChecker.exe` だけ追跡例外にする方針は妥当で、`git add build/nuitka/pdfDiffChecker.exe` は可能な状態にできる。
- **GitHub Releases のアセットは自動では付かない**。`gh release create` や Web UI でリリースを作るとき、**ローカルの `build\nuitka\pdfDiffChecker.exe` を引数で渡す／手動アップロード**しない限り、リポジトリに exe がコミットされていてもリリースページに exe は出ない。
- 例外ルールは **パターンの順序と `**` の範囲**で環境によって誤解されやすいため、意図が一目で分かるコメント付きパターンに整える。

## スコープ

- `docs/dev/RELEASE_CHECKLIST.md`（手順の明確化、ワンライナー例）。
- 任意: `scripts/` 配下に **タグと exe パスを渡して `gh release create` する補助スクリプト**。
- `.gitignore`（`build/` 例外の明示化）。
- `controllers/keyboard_navigation.py`（コンボの Down 自動 Post 削除、空間ナビとポップダウン開閉の両立）。
- `utils/preview_diff_emphasis.py` および `views/main_tab.py`（差分マスクの補強。既存のインク XOR フローを主とし、**両レイヤーにインクが重なる画素で RGBA が十分違う領域**を追加で強調するなど）。

## 非対象

- 新しいマイナーバージョンのタグ付けリリース作業そのもの（本マイルストーンは実装とドキュメントまで）。
- PDF 操作タブ／画像タブ専用の別アルゴリズム（必要なら後続マイルストーン）。

## 差分強調の方針（技術）

- 現状の `build_diff_highlight_overlay_rgba` は、インクマスクの膨張一致で **同じ太さの線上の色だけ違う**・**字形が部分的に置き換わった**領域を XOR から消しやすい。
- **補助マスク**: ベース・比較の両方でインクとみなせる画素のうち、**ピクセル単位の RGBA 二乗誤差が閾値を超える**ものを別途マスクし、既存の `base_only` / `comp_only` と重複しない部分だけオーバーレイに乗せる（パラメータで調整可能にする）。

## 検証手順（目安）

1. メインタブでコンボ（言語・DPI・色処理）にフォーカスを移し、**上下キー**で隣のコントロールへフォーカスが移ること。**Enter** でだけリストが開くこと（開いたあとは上下で項目移動できること）。
2. 小さな文字変更・細線の差分を含むサンプルで「差分を強調表示」ON 時、従来より強調が目立つこと（過剰に全面が赤くならないこと）。
3. `git check-ignore -q build/nuitka/pdfDiffChecker.exe` が「無視されていない」こと、`RELEASE_CHECKLIST` のコマンド例どおり `gh release create ... pdfDiffChecker.exe` でアセットが付くこと（実リリースは任意）。
