# M8 Plan — リリース資産、コンボボックス、差分強調、プレビュー UX

## 目的

1. **GitHub Release に exe が付かなかった理由**を整理し、再発しにくい手順・設定にする。
2. **キーボード**: コンボボックスにフォーカスがあるとき、**Enter を押すまで**矢印キーでプルダウンを開かず、上下キーで隣接コントロールへフォーカスを移せるようにする。
3. **差分強調**: 当初の M6 と同様、**画素差分＋軽いモルフォロジ**による**単一マスク**で、**二色化／指定色濃淡を分岐させない**統一ロジックに戻す（ノイズと可視性のバランスはユーザー検証で調整）。
4. **プレビュー UX**: 重い処理中に**プログレスバー**を表示する。変身（DPI 反映）後は**ページ番号を維持**する（数十ページ時の利便性）。

## 背景・原因整理（exe と .gitignore）

- `.gitignore` で `build/` を広く無視しつつ `build/nuitka/pdfDiffChecker.exe` だけ追跡例外にする方針は妥当で、`git add build/nuitka/pdfDiffChecker.exe` は可能な状態にできる。
- **GitHub Releases のアセットは自動では付かない**。`gh release create` や Web UI でリリースを作るとき、**ローカルの `build\nuitka\pdfDiffChecker.exe` を引数で渡す／手動アップロード**しない限り、リポジトリに exe がコミットされていてもリリースページに exe は出ない。
- 例外ルールは **パターンの順序と `**` の範囲**で環境によって誤解されやすいため、意図が一目で分かるコメント付きパターンに整える。

## スコープ

- `docs/dev/RELEASE_CHECKLIST.md`、`.gitignore`、`scripts/`（任意）、`controllers/keyboard_navigation.py`。
- `utils/preview_diff_emphasis.py`：**M6 初期の `rgba_pixel_diff_mask` ベース**の `build_diff_highlight_overlay_rgba`（インク XOR／same-cell 補助は廃止）。
- `views/main_tab.py`：差分オーバーレイは**処理済みラスタのみ**で上記を呼ぶ。プログレス表示と変身後の `current_page_index` 維持。

## 二値化（二色化）でノイズが出る理由（設計メモ）

- PDF→ラスタでは**アンチエイリアス**や圧縮により、理論上の「純粋な赤／青」ではなく**境界が中間色**になる。
- **二値化閾値**は RGB 合計などの単純規則のため、わずかな輝度差で片側だけインク扱いになる画素が残る。
- **位置合わせ・スケール**のサブピクセル差で、同じ線でも画素グリッド上のパターンがずれ、**画素差分**では常に「違い」として残りやすい。
- 「原色だけを厳密検出」するには、**HSV 範囲＋形態処理**や**ベクター比較**、別ライブラリによる**特徴マッチング**など、別アプローチの検討余地がある（後続マイルストーン候補）。

## 非対象

- 新しいマイナーバージョンのタグ付けリリース作業そのもの。
- PDF 操作タブ／画像タブ専用の別アルゴリズム（必要なら後続）。

## 検証手順（目安）

1. コンボの矢印／Enter 挙動（M8-B）。
2. 二色化・指定色濃淡の両方で差分強調が**同じ種類のオーバーレイ**になり、従来より意図に沿うか。
3. ページ移動・変身・色モード変更・差分 ON/OFF でプログレスが表示されること。
4. 変身後、変身前と同じページ番号（範囲内）に留まること。
5. `RELEASE_CHECKLIST` と `git check-ignore`（exe 追跡）の確認。
